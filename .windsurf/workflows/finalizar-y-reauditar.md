# Finalizar y reauditar

1. Ejecuta las validaciones directamente relacionadas con el cambio.
2. Comprueba `git diff --check`.
3. Revisa el diff una vez.
4. Confirma que no se modificó contenido fuera del alcance.
5. Confirma que no quedan temporales propios de la tarea.
6. Realiza commit, push o deploy solo cuando estén autorizados.
7. Comprueba el resultado real de las operaciones Stateful.
8. Actualiza el plan y BITACORA.md cuando corresponda.
9. Entrega un único informe final.
